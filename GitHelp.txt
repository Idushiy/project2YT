git status проверяет статус нынешнего проекта
git add добавляет изменения в проект
git commit сохраняет данные добавленные с помошью add в слепок 
git push добавляет локальные изменение (сравнивая чего не хватает) на удалённый сервер
git remote добавляет\удаляет удалённые репозитории
git diff показывает изменения в последнем коммите (--check)
git pull забирает новые изменения в удалённого репазитория, а затем сливает их с текущей веткой  
git clone клонирует репозитурий, создавая там рабочую директорию 
git init превращает обычную директорию в GIT дерикторию, присваивая имя ветки и отслеживая её
git reset ( "git reset ." - все) можно отменить команду add
git clean очищает директорию от файлов, от не отслеживаемых
git config изначальные настройки, вводится имя и мыло
git branch  просмотр наличие веток и создание
git cheackout взглянуть на указанную ветку 
git merge сливает ветки в текущую 
git stash используется для временного сохранения всех незакомминченых файлов
git tag создаёт матки в проекте, как важные места в проекте 
git log просмотр истории комитов 
git rebase Записывает последнии изменения Коммит из указанной ветки в текущую
git cherry-pick создаёт новый коммит и основе выбранного коммита, тоесть применяет все коментарии и сообщения именно его 
git rm удаляет файл из отслеживаемых и из рабочего каталога
git mv перемещение файлов по каталогам

fetch связывается с удалённым репазиторием и забирает все измен					 ения которых у нас нет


переименовать ветку git
git branch -m old-branch new-branch
git push -u origin new-branch
